# AWS VPC Network Traffic Anomaly Detection

A serverless machine learning solution that detects anomalous network traffic from Amazon VPC Flow Logs using AWS Lambda, Amazon CloudWatch Logs, Isolation Forest, and Amazon SNS.

---

- Amazon VPC Flow Logs
- Amazon CloudWatch Logs

## Workflow

1. Amazon VPC Flow Logs capture network traffic metadata.
2. CloudWatch Logs stores the flow log records.
3. AWS Lambda retrieves and parses the log events.
4. Network features are extracted from each flow.
5. Isolation Forest analyzes the extracted features.
6. Anomalous network flows are identified.
7. Anomaly scores are generated.
8. Anomalous flows are grouped by Source IP.
9. Amazon SNS sends an email notification containing the anomaly report.

## Network Features Analyzed

The model analyzes the following VPC Flow Log attributes:

- Source IP
- Destination IP
- Source Port
- Destination Port
- Protocol
- Packet Count
- Byte Count
- Traffic Action (ACCEPT / REJECT)

These features help identify statistically unusual network traffic patterns.

## Getting Started

### Clone Repository

```bash
git clone https://github.com/Priya-1602/aws-vpc-network-anomaly-detection.git
```

### Install Dependencies

```bash
pip install -r requirements.txt
```

### Deploy

1. Create an AWS Lambda function.
2. Upload the Python source code.
3. Install the required dependencies.
4. Enable Amazon VPC Flow Logs.
5. Configure an Amazon CloudWatch Log Group.
6. Create an Amazon SNS Topic.
7. Subscribe an email endpoint.
8. Configure IAM permissions for:
   - Amazon CloudWatch Logs
   - Amazon SNS
9. Execute the Lambda function.
